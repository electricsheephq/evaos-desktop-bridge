# Desktop Bridge Snapshot Schema

## Summary

This document defines the MVP JSON contract for `evaos-desktop-bridge`. The schema is intentionally small, redacted by default, and limited to visible desktop observation commands.

## Envelope

Every successful or failed command returns one JSON object:

```json
{
  "schema_version": "2026-05-02.mvp1",
  "command": "codex.snapshot",
  "target": "codex",
  "timestamp": "2026-05-02T00:00:00Z",
  "ok": true,
  "data": {},
  "warnings": [],
  "errors": [],
  "audit_id": "audit-0123456789abcdef"
}
```

Fields:

- `schema_version`: Stable MVP schema id.
- `command`: One of `status`, `codex.focus`, `codex.snapshot`, `codex.ax_tree`.
- `target`: `desktop` or `codex`.
- `timestamp`: UTC ISO-8601 timestamp generated by the bridge.
- `ok`: Boolean command outcome.
- `data`: Command-specific payload.
- `warnings`: Non-fatal issues, already redacted.
- `errors`: Structured fatal issues, already redacted.
- `audit_id`: Identifier matching the local append-only audit log record.

## Structured Errors

Errors use this shape:

```json
{
  "code": "permission_missing",
  "message": "Accessibility permission is required.",
  "guidance": "Open System Settings > Privacy & Security > Accessibility ...",
  "permission": "accessibility"
}
```

`permission` is present only when the error maps to a macOS permission.

## Command Data

`status` returns:

```json
{
  "platform": "Darwin",
  "app": {
    "name": "Codex Desktop",
    "process_name": "Codex",
    "installed": true,
    "running": true,
    "pid": 12345,
    "paths": ["~/Applications/Codex.app"]
  },
  "permissions": {
    "accessibility": {"status": "granted", "guidance": "..."},
    "screen_recording": {"status": "unknown", "guidance": "..."}
  },
  "safety": {
    "read_only": true,
    "sends_prompts": false,
    "uses_internal_mutation_rpc": false,
    "reads_session_databases": false
  }
}
```

`codex.focus` returns:

```json
{
  "focused": true
}
```

With `--dry-run`, it returns:

```json
{
  "would_focus": true,
  "focused": false
}
```

`codex.snapshot` returns:

```json
{
  "timestamp": "2026-05-02T00:00:00Z",
  "frontmost_app": "Codex",
  "window_title": "Codex",
  "screenshot_path": "~/Library/Application Support/evaos-desktop-bridge/screenshots/20260502T000000Z.png",
  "max_chars": 4000
}
```

`codex.ax_tree` returns:

```json
{
  "nodes": [
    {"role": "AXWindow", "name": "Codex"},
    {"role": "AXButton", "name": "New Chat"}
  ],
  "truncated": false,
  "max_nodes": 200
}
```

## Redaction and Caps

- Current user home paths are converted to `~`.
- Generic `/Users/<name>` paths are converted to `~`.
- Secret-like API keys, bearer tokens, and authorization headers are replaced with `<redacted-secret>`.
- Snapshot text is capped by `--max-chars`.
- AX tree output is capped by `--max-nodes` and includes roles/names only.

## Non-Goals

This schema does not include prompts, session transcripts, hidden Codex state, app-server RPC data, auth files, or database rows.
